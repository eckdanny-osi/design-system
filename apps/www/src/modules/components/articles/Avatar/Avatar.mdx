## Synopsis

`Avatar`s are visual representations of a user or other identifiable entity.

<DemoCard>
  <Avatar />
</DemoCard>

```jsx
import { Avatar } from '@cwds/components'
```

The `Avatar` manifests (in order of precedence) as:

1. a raster **image** (if available)
1. **text** initials (if available)
1. generic **icon** (fallback)

<div className="d-flex justify-content-around mb-3">
  <div className="text-center">

      <Avatar imgUrl="https://bit.ly/2Ar9w7z" alt="Beardy McBeardFace" />
    </DemoCard>
    <div>image</div>
  </div>
  <div className="text-center">
    <DemoCard className="p-2 mb-0">
      <Avatar text="WW" />
    </DemoCard>
    <div>text</div>
  </div>
  <div className="text-center">
    <DemoCard className="p-2 mb-0">
      <Avatar />
    </DemoCard>
    <div>icon</div>
  </div>
</div>

```jsx
<Avatar imgUrl="https://bit.ly/2Ar9w7z" alt="Beardy McBeardFace" />
<Avatar text="WW" />
<Avatar />
```

## Usage Guidelines

Use the best available `Avatar` type for optimal recognition with the user

Use an appropriate `size` based on placement. (See [Adjusting the size value](#adjusting-the-size-value)).

### Accessibility

Do provide meaningful <code>alt</code> text when using **image** or **icon** `Avatar` types as necessary (see [WebAIM: Alternative Text](https://webaim.org/techniques/alttext/)).

## Examples

The default `Avatar` (with no additional props provided) uses an [`Icon`](icon) to represent an ambiguous identity:

<div className="mb-3">
  <Avatar />
</div>

```jsx
<Avatar />
```

### Adjusting the size value

The `size` can be specified using one of the Design System's <Link to="/core-style">discrete size values</Link>:

<div className="mb-3">
  <Avatar size="xs" /> <Avatar size="sm" /> <Avatar size="md" />{' '}
  <Avatar size="lg" /> <Avatar size="xl" />
</div>

```jsx
<Avatar size="xs" />
<Avatar size="sm" />
<Avatar size="md" />
<Avatar size="lg" />
<Avatar size="xl" />
```

<!-- @todo(mj): what are these unknown things? -->

- `xs` should be used for [on-screen metadata]()
- `sm` should be used in small areas like 40px text fields
- `md` should be used for [activity streams](), [comments](), or [table views]() for [project listings]()
- `lg` should be used in [`PageTitles`](), like a user's account settings
- `xl` should be used in prominent positions on the [`Page`](), like in [`UserProfile`]()

### Inverting

The color theme can be easily inverted for contrast flexibility:

<div className="mb-3">
  <Avatar invert />
</div>

```jsx
<Avatar invert />
```

### Using Initials

When user or identifiable entity does not have an `imgUrl` (path to a raster graphic), use the `text` prop instead:

<div className="mb-3">
  <Avatar text="DE" />
</div>

```jsx
<Avatar size="lg" text="DE" />
```

### Using Images

Use this form whenever graphic media is available:

<div className="mb-3">
  <Avatar imgUrl="https://bit.ly/2Ar9w7z" size="lg" />
</div>

```jsx
<Avatar imgUrl="user.jpg" size="lg" />
```

<Alert color="warning">

  <code>size</code> less than <code>lg</code>!
